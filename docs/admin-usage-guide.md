# @blueSky 管理画面 使い方ガイド

> オーナー向け・各機能の使い方
> 障害対応・インフラ運用は「📘 運用手順書」をご覧ください

---

## 1. ダッシュボード（予約カレンダー）

左サイドバーの「📅 予約カレンダー」をタップ。

- 月単位のカレンダー表示で予約状況を一覧
- 各日付の色: 緑=確定 / 黄=確認中 / 赤=キャンセル
- 「未承認の抽出案」バッジが上部に出ているときは LINE で受けた注文の AI 抽出案があります → タップして承認

---

## 2. 予約・お客様対応

### 📋 予約一覧

サイドバー「予約・お客様」>「📋 予約一覧」

#### 検索・絞り込み

- **キーワード**: 名前・メール・電話で部分一致検索
- **ステータス**: 確認中 / 確定 / キャンセル
- **宿泊タイプ**
- **期間**: チェックイン日の範囲

#### CSV ダウンロード

右上の「📥 CSVダウンロード」ボタンで、**絞り込んだ条件のまま**Excel/Numbers 用CSV を出力できます。
税理士への共有や月次集計に使えます。

#### 個別予約の編集

予約をタップ → 詳細画面で金額やオプションを修正可能。

> ⚠️ 「manualAmount」を ON にすると自動再計算されません。割引適用時に活用してください。

### 📝 抽出案承認（LINE 連携）

サイドバー「予約・お客様」>「📝 抽出案承認」

お客様が LINE で送信した注文を AI が抽出して保存しています。
内容を確認して「承認」または「拒否」してください。

- **承認** → sale_lines に登録され、在庫が減り、売上仕訳が自動作成されます
- **拒否** → 理由を任意で入力（AI 精度改善データとして残ります）
- **編集** → 商品・数量・単価・日付をその場で修正してから承認できます

> 💡 信頼度（confidence）が緑色（70%以上）は基本そのまま承認OK、黄・赤は要チェック。

### 🚫 日程ブロック

サイドバー「予約・お客様」>「🚫 日程ブロック」

定休日・私用などで予約を受け付けない日を設定します。
カレンダーで「×」表示になり、お客様は予約できなくなります。

## 3. 販売・在庫

### 🍖 商品・メニュー管理

商品マスタを編集します。

- **販売可** (`is_sellable`): お客様への販売対象（ビール等）
- **在庫追跡** (`track_inventory`): 在庫数を管理する商品
- **販売価格** / **原価**: 売上仕訳・棚卸計算に使用

### 📦 在庫管理

- 現在在庫数の確認
- **棚卸**: 期末・期首の在庫スナップショットを保存
- **入庫記録**: 仕入時に登録

### 🎒 レンタル管理

テーブル・椅子・調理器具など、予約時に追加できるレンタル品の管理。

### 💴 料金設定

#### 基本料金

宿泊タイプごとの1泊料金を編集。

#### 割引・季節料金

- **連泊割引**: 2泊目以降の割引率（例: 10%）
- **季節料金**: 期間別の倍率（例: 夏休み +20%、閑散期 -20%）

→ お客様の確認画面で自動適用されます。

## 4. 会計

### 🧮 仕訳・元帳

- 自動仕訳の確認・編集
- 総勘定元帳の閲覧
- 試算表で借方・貸方の整合性確認

### 📊 決算書

サイドバー「会計」>「📊 決算書」

- **損益計算書 (P/L)**: 売上・経費・所得金額
- **貸借対照表 (B/S)**: 資産・負債・資本

#### 期間切替

- **年度プルダウン**: 過去5年〜来年
- **月次 / 年次タブ**: 月単位 or 年度全体

#### CSV エクスポート（青色申告用）

- **📥 弥生CSV**（Shift_JIS）: 弥生青色申告に取り込み可
- **📥 freee CSV**（UTF-8 BOM）: freee に取り込み可

→ 取り込み先で正式書式の決算書PDFを出力 → e-Tax で電子申告

## 6. その他

### ⚙️ サイト設定

サイト全体に影響する基本情報:

- **チェックイン時間** / **チェックアウト時間**
- **所在地** / **緊急連絡先**
- **ご利用案内**（持ち物・施設・チェックイン手順など）
- **アクセス案内**（電車・車・周辺観光）

→ 予約確認メール、領収書PDF、`/access` ページ、ホーム Access セクションに自動反映されます。

### 📘 運用手順書

インフラ・障害対応・バックアップ・コスト管理など、技術運用の手順書。

## 7. チェックイン当日のフロー

1. お客様が照会ページで QR コードを表示
2. オーナーがスマホで QR をスキャン
3. `/admin/checkin/[予約ID]` が開く
4. 予約内容を確認
5. **「✅ チェックイン完了として記録」** ボタンをタップ

→ `reservations.checked_in_at` に時刻が記録されます。
