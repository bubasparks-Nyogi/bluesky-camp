# Phase 16 Performance & E2E Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development

**Goal:** Final polish before declaring v1 done. Tighten image loading, audit queries that fetch `*` where smaller selects work, and add Playwright smoke tests for the public-facing critical paths so we can refactor safely going forward.

**Architecture:**
- Replace direct `<img>` tags in our own components with Next.js `<Image>` where it pays off (cover images, gallery, fixed-size hero shots). Skip `<img>` inside markdown (rendered by react-markdown via custom `img` component — leaves runtime control).
- Audit `supabase ... .select('*')` on the home page's hot path and trim where reasonable.
- Add Playwright with three smoke tests against the public site. Run via `npx playwright test`. Not added to CI yet — owner runs manually.

**Tech Stack:** Next.js 14 `next/image`, Playwright, TypeScript, existing TailwindCSS.

(skip PostBody — that's markdown rendering with arbitrary user content; `<img>` is fine there)

- [ ] **Step 2: For each `<img>` you find in your own components, convert to `<Image>`**

Pattern:
```tsx
import Image from 'next/image'
// before
<img src={p.cover_image} alt={p.title} className="w-full h-40 object-cover" />
// after — if dimensions known
<Image src={p.cover_image} alt={p.title} width={400} height={160} className="w-full h-40 object-cover" />
// after — if fill needed
<div className="relative w-full h-40">
  <Image src={p.cover_image} alt={p.title} fill className="object-cover" />
</div>
```

For images from external URLs (e.g., user-supplied `cover_image` URLs hosted on Supabase storage or anywhere), make sure `next.config.mjs` allows the hostname. Read `next.config.mjs` first.

Add to `next.config.mjs`:
```javascript
images: {
  remotePatterns: [
    { protocol: 'https', hostname: 'frdiafkdjeaslhwlvfxa.supabase.co' },
    { protocol: 'https', hostname: '*.supabase.co' },
  ],
},
```
(only if not already present)

- [ ] **Step 3: Create `e2e/home.spec.ts`**

```typescript
import { test, expect } from '@playwright/test'

test.describe('Home page', () => {
  test('renders main sections', async ({ page }) => {
    await page.goto('/')
    await expect(page).toHaveTitle(/blueSky/i)
    // Hero / brand visible
    await expect(page.locator('text=@blueSky').first()).toBeVisible()
    // Calendar section heading
    await expect(page.getByRole('heading', { level: 2 }).filter({ hasText: /カレンダー|予約状況/ })).toBeVisible({ timeout: 10_000 })
    // Reviews heading
    await expect(page.getByRole('heading', { level: 2 }).filter({ hasText: /クチコミ/ })).toBeVisible()
  })
})
```

- [ ] **Step 4: Create `e2e/reserve.spec.ts`**

```typescript
import { test, expect } from '@playwright/test'

test.describe('Reservation flow', () => {
  test('reserve page loads with step 1 (date selection)', async ({ page }) => {
    await page.goto('/reserve')
    await expect(page).toHaveURL(/\/reserve/)
    // Step indicator or date pickers should be present
    await expect(page.locator('input[type="date"]').first()).toBeVisible({ timeout: 10_000 })
  })

  test('reserve page accepts date param and skips to step 2', async ({ page }) => {
    await page.goto('/reserve?date=2027-01-15')
    // Date field should be pre-filled
    const dateInputs = page.locator('input[type="date"]')
    await expect(dateInputs.first()).toHaveValue('2027-01-15', { timeout: 10_000 })
  })
})
```
